# Тестовое задание для стажёра Frontend-разработчика (CDEK)

## О задании

Мы в **CDEK** постоянно работаем над улучшением внутренних сервисов для сотрудников. Один из ключевых сервисов - **новостной портал**, где публикуются важные анонсы, обучающие материалы, акции и события из жизни компании.

## Задача

Разработать **универсальный компонент новостного блока**, который будет использоваться на нескольких страницах новостного портала CDEK. Компонент должен получать данные с существующего API и отображать новости с пагинацией.

## Макет

На макете представлен компонент в нескольких состояниях ("Новости компании" - у каждой новости видно картинку, "Рубрика Бизнес" - картинка только у первой и пустой компонент когда в рубрике нет новостей). Также представлено мобильное отображение.

### Функциональные требования

1. **Загрузка новостей**
   - При монтировании компонента загружается первая страница новостей
   - Во время загрузки показывается индикатор (skeleton или спиннер)
   - Если новостей нет, показываем плейсхолдер

2. **Отображение новости**
   - Должно соответствовать макету

3. **Пагинация**
   - Кнопки "Назад"/"Вперёд"
   - При переходе на другую страницу данные перезагружаются
   - Если страница последняя — кнопка "Вперёд" неактивна
   - Если страница первая — кнопка "Назад" неактивна

4. **Адаптивность**
   - На мобильном устройстве компонени должен подстраиваться под ширину экрана

5. **Обработка ошибок**
   - Если API недоступно — показать сообщение об ошибке
   - Если новостей нет — показать плейсхолдер

### Дополнительные требования (будет плюсом)

- **Кэширование** — загруженные страницы не запрашиваются повторно
- **Ленивая загрузка изображений** — обложки подгружаются при скролле
- **Анимации** — плавное появление новостей
- **Типизация** — использование TypeScript

## Технологический стек

**Обязательно:** React

**Можно использовать:**
- **React:** Next.js, Vite + React, Create React App

**Стилизация:** CSS Modules, Styled Components, Tailwind CSS, SCSS (на выбор)

**Набор иконок:** мы используем Tabler Icons во всех проектах

## В каком виде представить работу

В виде React-приложения, которое можно запустить локально, но будет плюсом, если у вас получится задеплоить его и дать ссылку на работающую версию. На странице должно быть несколько вариантов получившегося компонента (Новости компании, Новости рубрики Бизнес и пустой вариант). В ПК-версии разместить их можно например по центру страницы с нейтральным фоном, но творческий подход оценим :). Не забудьте протестировать мобильную версию (большинство наших пользователей заходят с мобильных устройств)

## Критерии оценки

1. **Работоспособность** — всё работает согласно ТЗ
2. **Качество кода** — читаемость, структура, отсутствие дублирования
3. **UX/UI** — удобство использования, обработка состояний загрузки/ошибок
4. **Адаптивность** — корректное отображение на всех устройствах
5. **Дополнительные требования** — реализованные плюшки (на ваше усмотрение, выделят вас среди конкурентов)
